# Task 7.2: Effort vs Impact Prioritization Matrix

## Overview

This document prioritizes all cleanup and addition work items by their effort-to-impact ratio, enabling optimal resource allocation during the dashboard-only migration. Items are scored and ranked to identify "quick wins" (low effort, high impact) and "strategic investments" (high effort, high impact).

---

## Scoring Methodology

### Impact Score (1-10)

| Score | Description |
|-------|-------------|
| 10 | Migration blocker - cannot proceed without this |
| 8-9 | Critical workflow - significant user degradation |
| 6-7 | Important - noticeable impact, workarounds painful |
| 4-5 | Useful - moderate impact, acceptable workarounds |
| 2-3 | Nice-to-have - minimal user impact |
| 1 | Negligible - developer-only or infrequent use |

### Effort Score (1-10)

| Score | Description | Time Estimate |
|-------|-------------|---------------|
| 1 | Trivial | < 1 hour |
| 2-3 | Low | 1-4 hours |
| 4-5 | Medium | 1-2 days |
| 6-7 | High | 3-5 days |
| 8-9 | Very High | 1-2 weeks |
| 10 | Massive | > 2 weeks |

### Priority Formula

```
Priority Score = Impact Score / Effort Score × 10
```

Higher scores = better value for investment.
