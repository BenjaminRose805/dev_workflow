## Integration Details

### 1. Shell → yazi
| Type | Description |
|------|-------------|
| Function | `y()` wrapper enables cd-on-exit |
| Mechanism | Writes cwd to temp file, shell reads and cd's |

### 2. yazi → glow
| Type | Description |
|------|-------------|
| Plugin | `glow.yazi` for markdown preview |
| Config | `prepend_previewers = [{ name = "*.md", run = "glow" }]` |
| Trigger | Automatic on .md file selection |

### 3. yazi → git
| Type | Description |
|------|-------------|
| Plugin | `git.yazi` for status indicators |
| Config | `require("git"):setup()` in init.lua |
| Display | Modified/staged/untracked indicators in file list |

### 4. yazi → micro
| Type | Description |
|------|-------------|
| Mechanism | $EDITOR environment variable |
| Trigger | Press Enter on file in yazi |
| Fallback | Uses system default if EDITOR unset |

### 5. micro → fzf
| Type | Description |
|------|-------------|
| Plugin | fzf plugin (main.lua) |
| Command | `:fzf` in micro |
| Action | Opens fzf, selected file opened in buffer |

### 6. micro → LSP (pylsp)
| Type | Description |
|------|-------------|
| Plugin | lsp plugin |
| Config | `lsp.server = "python=pylsp"` |
| Features | Completion, hover, go-to-def, format |

### 7. gh → git
| Type | Description |
|------|-------------|
| Built-in | gh wraps git commands |
| Alias | `gh co` = `gh pr checkout` |
